Hiện tại:

Reset → Worker lưu data cũ vào KV → Client tự phát hiện → Client tự retry mãi
Cách mới:

Reset → Worker thử lấy data → data cũ → Worker tự chờ 30s → thử lại → data mới → lưu KV → Client fetch về
Tức là chuyển toàn bộ logic chờ từ client lên Worker. Client chỉ poll đơn giản, không cần biết gì thêm.

Trade-off:

Hiện tại	Cách mới
Worker execution time	~1s/lần	~2-5 phút/lần khi retry
Cloudflare free limit	10ms CPU/request	Worker timeout 30s — nếu retry quá lâu bị kill
Client code	Phức tạp, dễ bug	Đơn giản, ít bug

Hiện tại mỗi user:

Fast poll: 8 request × N users
Slow poll: 5 request × N users
100 users = 1300 request mỗi lần reset

Cách mới mỗi user:

Poll /updated mỗi 30s → nhẹ hơn, chỉ trả timestamp string
Thấy đổi → 1 request fetch /stock
100 users = ~100 request nhỏ + 100 request lớn

Giảm ~10x request, và /updated response nhỏ hơn /stock nhiều lần`
